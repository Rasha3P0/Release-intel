Title: Tool approval guardrails for GitLab Duo agents (beta)
Release: 19.1
Source: https://docs.gitlab.com/releases/19/gitlab-19-1-released/
Tier: Premium, Ultimate
Offering: GitLab.com, GitLab Self-Managed, GitLab Dedicated, GitLab Dedicated for Government
Links: Documentation · Related Issue

Administrators can now configure tool-level approval policies for GitLab Duo agents, gating sensitive actions with human
approval at the moment of execution.
Previously, after an AI agent was approved for a project, it could invoke any of its tools without further review, including write and
destructive operations. Now, you can define rules for groups and projects that map each tool to one of three modes:
Allow (execute silently).
Ask (require human approval).
Deny (block entirely).
When an AI agent calls a tool in “askˮ mode, the user is prompted with an inline approval card before execution proceeds.
This beta release includes Agentic Chat, IDE, and flows, and emits audit events for every approval decision.
