Title: Improved secret detection coverage for feature branch pipelines
Release: 19.1
Source: https://docs.gitlab.com/releases/19/gitlab-19-1-released/
Tier: Free, Premium, Ultimate
Offering: GitLab.com, GitLab Self-Managed, GitLab Dedicated, GitLab Dedicated for Government
Links: Documentation · Related Issue

In GitLab versions earlier than 19.1, you couldnʼt trust a feature branch pipeline to surface every secret in your branch. A newbranch scanned only the latest commit. An existing branch scanned only your most recent push. A credential leaked in an earlier
commit could sit undetected, reaching shared branches or production before being flagged.
Now you can catch those secrets where theyʼre cheapest to fix. In GitLab 19.1, secret detection scans every commit from the
branchʼs divergence point with the default branch to the latest commit. That means fewer secrets slip through to later stages,
less time rotating exposed credentials after the fact, and consistent, predictable coverage across your branches.
