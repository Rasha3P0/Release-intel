Title: Close coverage gaps with the Scanner Enablement Wizard
Release: 19.1
Source: https://docs.gitlab.com/releases/19/gitlab-19-1-released/
Category: Unified DevOps and Security
Tier: Ultimate
Offering: GitLab.com, GitLab Self-Managed, GitLab Dedicated
Links: Documentation · Related Issue

You can now use the Scanner Enablement Wizard to close scanner coverage gaps across your projects without manually
identifying which projects need attention.
Security configuration profiles define which scanners run and how. Security inventory shows scanner coverage across your
projects and lets you bulk-apply profiles to selected projects or subgroups. The wizard adds a goal-driven workflow on top: you
set the goal, and it finds the projects missing coverage and closes only those gaps.
